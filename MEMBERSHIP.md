# MEMBERSHIP.md — 마실맵 ↔ 통합 회원구조 연동 메모

> 기준: MasilMAP ver2 **rel2** · 2026-05-30
> 이 파일은 프론트 번들에 동봉하는 회원구조 연동 메모입니다. (HANDOFF.md의 짝)
> 단일 소스: 회원구조 규칙은 "마실그라운드 통합 회원 구조 설계도 v2"를 따릅니다.

---

## 왜 이 메모가 있나

마실맵은 마실그라운드·언빌트와 **하나의 회원 정체성**을 공유합니다. 목적은 통합 로그인이 아니라,
사람의 이력과 그에 따라붙는 권리(로열티)를 끊김 없이 추적하는 것입니다. 프론트를 만들 때 아래
3가지 약속을 지키면, 나중에 백엔드를 붙일 때 화면을 다시 짤 필요가 없습니다.

---

## 지켜야 할 3가지 약속

### 1) 사람은 user_id로 묶는다 (이름·이메일 아님)
- 표시는 이름/handle로 하되, 내부 참조·연결은 불변 식별자 `userId`로.
- 이메일·SNS는 바뀔 수 있는 "연락처 속성". 이메일이 바뀌어도 이력이 끊기면 안 됨.

### 2) mock 데이터 모양 = 백엔드 스키마 모양
- `data.jsx`의 픽스처를 "API 응답 계약"으로 본다. 모양을 맞춰두면 나중에 갈아끼우기만 하면 됨.
- 특히 권리 배분(split)은 문자열이 아니라 구조화 배열로 (아래 예시).

### 3) 공/사 격리
- 마이페이지(찜·후기·스탬프·예약)는 사적 영역. **소속·회사 정보를 절대 표시/전송하지 않는다.**
- 한 계정이 admin/tour/editor를 겸해도, 사적 활동은 역할 콘솔에서 보이지 않는다.

---

## rel2에서 이미 잘 되어 있는 것 (유지)

- 권리 게이트(`upload.jsx`): 공동 권리자·동의·보류 흐름 = 설계도의 Beneficiary/consent와 일치.
- `ASSETS`에 owner·consent·split 존재 = 정산 약정 개념 반영.
- 역할 분기(`__masilRole`: user/admin/tour/editor) = 서비스 역할 축.
- 업로드 동의에 "마실그라운드 제휴 노출", "데이터 이전 사전 통지" = 생태계 연동 의식.

## rel2에서 보강할 것 (다음 작업 때)

- USER·ASSETS·BOOKINGS에 `userId`/`ownerRef` 추가, 이름 문자열 참조 제거.
- split 문자열 → beneficiaries 배열로 구조화.
- 역할 진입을 URL 분리(데모)에서 실제 인증·권한 게이트로.
- 저장(찜)을 localStorage → 로그인 시 user_id 기준 서버 동기화(비로그인은 병합).

## 권장 mock 구조 (백엔드와 동일 모양)

```js
// USER — userId를 식별자로
const USER = {
  userId: "u_minseo",          // 불변 식별자 (신규)
  name: "민서",                 // 표시용
  handle: "@minseo",            // 표시/연락 속성
  // ... 기존 필드 유지
};

// ASSET — split을 구조화 배열로
{
  id: "A-3203",
  building: "whanki",
  name: "환기미술관_3D모델_v2.glb",
  type: "3D",
  registeredBy: "u_woo",        // userId
  beneficiaries: [
    { ownerRef: "u_woo",   role: "설계자", sharePct: 50, consentStatus: "보류" },
    { ownerRef: "org_woo", role: "협력",   sharePct: 30, consentStatus: "대기" },
    { ownerRef: "masilmap", role: "플랫폼", sharePct: 20, consentStatus: "완료" }
  ],
  status: "보류"                // 전원 consentStatus=완료 시 "공개"
}
```

표시용 문자열("설계자 50% · 협력 30% …")은 이 배열에서 만들어 쓴다.

## 화면별 연동 포인트 (요약)

| 화면 | 파일 | 연동 시 핵심 |
|------|------|--------------|
| 로그인/온보딩 | `auth.jsx` | OAuth → user_id 연결, 이메일은 속성, 본인인증 비강제 |
| 마이페이지 | `mypage.jsx` | 사적 활동, 소속/회사 미노출(격리), user_id 참조 |
| 권리 게이트 | `upload.jsx` | split 구조화, 권리자=id, 전원 동의 트랜잭션 |
| 권리 관리/콘솔 | `admin.jsx`·`consoles.jsx` | ASSETS 서버화, 역할+직급 권한 게이트 |
| 픽스처 | `data.jsx` | userId/ownerRef 추가, API 응답 견본화 |
