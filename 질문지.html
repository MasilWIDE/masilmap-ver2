실제 코드 오류 즉시 수정 필요
작동은 되지만 실제 데이터 연결이 없거나 깨진 부분
Q 01
내 마실(마이페이지)에 저장된 공간/코스가 아직 BUILDINGS.slice(0,6)으로 하드코딩되어 있습니다. 어떻게 처리할까요?
현재 로그인 여부와 관계없이 건축물 목록 첫 6개가 항상 "찜한 공간"으로 표시됩니다. mypage.jsx:8-9
Q 02
공간 상세 → 예약하기 → 결제 완료 후, 어디로 이동해야 하나요?
현재 결제 완료 버튼이 눌리면 특별한 처리 없이 그냥 화면이 끝납니다. 예약 내역이 마이페이지에 반영되지 않습니다. booking.jsx:250, 440
Q 03
공간 지도(mapmenu)에서 외부 지도 연결 링크(네이버맵, 카카오맵)가 데이터에 없어서 표시가 안 됩니다. 어떻게 할까요?
각 공간에 extMaps: [] 필드가 없으면 외부 지도 버튼이 아예 렌더링되지 않습니다. mapmenu.jsx:76-108
Q 04
공간 상세 페이지의 "노트/메모" 기능이 샘플 텍스트로 하드코딩되어 있습니다.
에디터 노트가 SAMPLE_NOTES라는 임시 데이터로 표시됩니다. 실제로 노트 작성/저장이 되지 않습니다. detail.jsx:147-151
Q 05
걷기 코스 인증(스탬프) 화면에서 GPS 확인이 1.4초 타이머로만 처리됩니다. 실제 위치 검증이 없습니다.
현장에 없어도 버튼을 누르면 방문 인증이 됩니다. walkcourse.jsx:10-12
Q 06
공간 업로드(등록) 화면에서 주소/좌표가 하드코딩된 예시값으로 표시됩니다. 지도 위치 지정 기능이 없습니다.
서울시 종로구 율곡로 83 인근 / 37.5775°N · 126.9869°E가 항상 고정 표시됩니다. upload.jsx:141

UX/UI 흐름 결정 설계 방향 필요
사용자 경험에 직접 영향을 주는 판단이 필요한 부분
Q 07
로그인을 안 한 사용자에게 어디까지 허용할까요?
현재는 "이어 걷기" 섹션만 로그인 없이 숨겨집니다. 찜, 리뷰 작성, 코스 등록, 예약 등이 로그인 없이 진입하면 어떻게 되는지 정의가 없습니다.
Q 08
로그인 화면으로 이동하는 시점을 어떻게 할까요?
지금은 상단 "로그인" 버튼을 눌러야만 로그인 화면으로 갑니다. 찜 버튼, 예약 버튼 등을 비로그인 상태에서 누르면 어떻게 되는지 처리가 없습니다.
Q 09
공간 목록(탐색) 화면의 필터 메뉴에 "준비 중"인 카테고리들을 어떻게 표시할까요?
"인테리어", "계획안", "여행지" 카테고리가 비활성(회색)으로 표시되며 "※ 다른 카테고리는 준비중" 텍스트가 보입니다. home.jsx:331-334
Q 10
공간 상세 화면에서 "이 공간이 포함된 코스" 섹션이 해당 코스가 없으면 아무것도 표시되지 않습니다.
데이터가 없는 공간은 관련 코스 영역이 완전히 사라져서 레이아웃이 이상해질 수 있습니다.
Q 11
메인 페이지에서 검색 후 결과가 없을 때 어떻게 보여줄까요?
현재 "선택한 필터에 해당하는 공간이 없습니다"만 표시됩니다. 검색어나 필터를 바꾸라는 안내가 없습니다.
Q 12
공간 목록 화면의 정렬 기준을 어떻게 할까요?
현재 "↕ 정렬" 버튼이 있지만 실제로 아무 기능이 없습니다. home.jsx:1098
Q 13
네비게이션 바에 "코스"와 "시리즈"가 따로 있는데, 일반 사용자에게 두 개념의 차이가 명확하지 않을 수 있습니다.
코스 = 걷는 경로, 시리즈 = 여러 코스 묶음. 하지만 탭만 봐서는 차이를 알기 어렵습니다.
Q 14
상단 네비바의 검색창과 메인 페이지 중앙 검색창, 두 개가 동시에 존재합니다. 어색하지 않을까요?
메인 페이지에만 있는 구글형 큰 검색창 + 모든 페이지 상단 작은 검색창. 스크롤하면 큰 검색창이 사라지고 상단 작은 검색창만 남습니다.

Q 15
실제 결제 기능을 언제 붙일 계획인가요?
현재 "토스페이먼츠 결제 진행" 버튼이 있지만 실제 SDK 연결이 없고 시뮬레이션입니다. booking.jsx
Q 16
마실 앱(모바일)과 마실맵(웹)의 걷기 기록(doneStops)을 어떻게 동기화할 계획인가요?
현재 웹에서는 localStorage에 방문 기록을 저장합니다. 실제 모바일 앱과 연동하려면 서버 API가 필요합니다.
Q 17
공간 등록(업로드) 기능을 누구에게 열어줄 계획인가요?
현재 에디터/관리자 계정에만 업로드 화면이 있습니다. 일반 사용자의 공간 제보를 받을지 여부.
Q 18
피드(Feed) 화면 — 채널 팔로우 기능이 있는데, 이게 현재 MVP에 꼭 필요한가요?
채널 시스템, 팔로우, 채널별 코스 등이 연결되어 있어서 데이터 모델이 복잡합니다. 단순화할 수 있습니다.
Q 19
공간 사진 업로드 기능이 UI에는 있지만 실제로 작동하지 않습니다. 언제 구현할까요?
업로드 버튼이 있지만 파일 선택 후 아무 일도 안 일어납니다. Supabase Storage나 S3 연결이 필요합니다.
Q 20
권리 배분(저작권/로열티) 시스템을 코드에 구조는 있는데 UI가 없습니다. 필요한가요?
CLAUDE.md에 설계는 있지만 실제 화면이 없습니다. 설계자/사진가가 수익을 나눠받는 기능입니다.

Q 25
메인 페이지 "이번엔 어디로 마실 갈까요?" 섹션(기분/상황 칩)에서, 칩을 눌러도 코스 결과가 나오지 않는 경우가 있습니다. 어떻게 할까요?
데이터에 해당 기분(mood)에 맞는 코스가 없으면 "준비 중" 메시지가 뜹니다. 현재 실제 연결된 코스가 적습니다.
Q 26
공간 이미지 자리(ImgPlaceholder)가 현재 모두 회색 박스입니다. 실제 이미지가 들어올 때까지 어떻게 처리할까요?
이미지 URL이 없는 공간은 모두 비어 있는 것처럼 보입니다.
Q 27
모바일에서 떠다니는 버블 대신 가로 스크롤 칩이 표시되는데, 실제 폰에서 어떻게 보이는지 확인하셨나요?
메인 페이지 검색창 아래에 6개 칩이 가로 스크롤로 나타납니다. 실제 기기 테스트가 필요합니다.
